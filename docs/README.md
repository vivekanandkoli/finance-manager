## 📝 Documentation Guidelines

### Writing New Documentation

When adding new documentation:

1. **Choose the right folder:**
   - `guides/` - User-facing guides and tutorials
   - `reports/` - Progress reports, roadmaps, and status updates
   - `archived/` - Outdated documentation (for reference)

2. **Use clear titles:**
   - Use descriptive filenames in UPPERCASE_WITH_UNDERSCORES.md
   - Start with appropriate emoji in the title

3. **Follow the structure:**
   ```markdown
   # Title with emoji
   
   Brief description
   
   ## Section 1
   Content...
   
   ## Section 2
   Content...
   ```

4. **Keep it updated:**
   - Remove outdated information
   - Update links when files move
   - Archive old documentation instead of deleting
